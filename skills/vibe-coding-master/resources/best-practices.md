# Vibe Coding Best Practices

## The Golden Rules

1.  **Vibe over Syntax, Logic over Vibes.**
    -   Pide a la IA con "Vibe" (ej. "Hazlo minimalista"), pero verifica la lógica con rigor (Tests).

2.  **The Iron Law (TDD).**
    -   Nunca aceptes código si no has visto fallar un test primero.
    -   Si la IA escribe el código sin test, bórralo. "No Production Code Without Failure".

3.  **Clean Code for Humans.**
    -   Si tú no puedes leerlo, no sirve.
    -   Pide a la IA que refactorice hasta que sea legible como un libro.

4.  **Exocortex Mindset.**
    -   Todo lo que aprendes debe ser registrado (`symbiosis_log.md`).
    -   No eres un usuario, eres un entrenador de tu futuro yo digital.

5.  **Technical English Protocol.**
    -   Concepto en español -> Término en Inglés -> Inmersión total.
    -   No digas "la cosa esa que gira", di "Loop".

-   **Anti-Patterns:** Qué NO hacer (muy importante).
-   **Patterns:** La solución probada.
